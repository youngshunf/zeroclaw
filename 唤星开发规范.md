# 唤星 ZeroClaw 开发规范

> 版本：1.0
> 更新：2026-03-19
> 适用范围：huanxing-zeroclaw 仓库所有开发者

## 核心原则

**一句话总结：上游文件只加 `#[cfg(feature = "huanxing")]`，所有唤星逻辑都在独立目录。**

```
┌─────────────────────────────────────────────────┐
│ Layer 3: 唤星桌面端 (clients/desktop/)           │ ← 纯唤星代码，无冲突
│   Tauri + React + Sidecar 管理                   │
├─────────────────────────────────────────────────┤
│ Layer 2: 唤星扩展层 (src/huanxing/)              │ ← feature-gated 独立模块
│   Config扩展 + 路由扩展 + 渠道扩展               │
├─────────────────────────────────────────────────┤
│ Layer 1: ZeroClaw Core (src/*)                   │ ← 尽量不改，保持与上游同步
│   Agent + Providers + Tools + Gateway            │
└─────────────────────────────────────────────────┘
```

## 目录结构规范

```
huanxing-zeroclaw/
├── src/                          # ZeroClaw 核心（尽量不改）

│   │   ├── mod.rs                # 模块入口
│   │   ├── config.rs             # HuanxingExtConfig
│   │   ├── router.rs             # 唤星路由集合
│   │   ├── ws.rs                 # 唤星 WS handler
│   │   ├── cli.rs                # 唤星 CLI 子命令
│   │   ├── agents/               # Agent 管理
│   │   ├── sessions/             # 会话管理
│   │   └── channels/             # 唤星特有渠道
│   └── ...                       # 上游代码
├── crates/
│   └── hasn-client-core/         # HASN 协议客户端库
├── clients/desktop/              # 唤星桌面端（纯唤星代码）
│   ├── src/                      # 桌面端前端源码
│   │   ├── huanxing/             # 唤星特有组件
│   │   ├── components/           # 通用组件
│   │   └── ...
│   └── src-tauri/                # Tauri Rust 后端
└── web/                          # ZeroClaw 原版 Web（不改）
    └── src/huanxing/             # 唤星前端扩展（已废弃，用 clients/desktop/）
```

## 代码修改规范

### 规则 1：上游文件只加 feature gate

**允许**：
```rust
// src/config/schema.rs — 只加一行
#[cfg(feature = "huanxing")]
#[serde(default)]
pub huanxing: crate::huanxing::HuanxingExtConfig,
```

**禁止**：
```rust
// ❌ 不要在上游 struct 里直接加字段
pub struct Config {
    pub title_model: String,        // ❌ 唤星特有字段
    pub provider_api: ProviderApi,  // ❌ 唤星特有字段
}
```

### 规则 2：路由用 merge，不逐行添加

**允许**（gateway/mod.rs 只加一行）：
```rust
#[cfg(feature = "huanxing")]
let app = app.merge(crate::huanxing::router::huanxing_routes(shared_state.clone()));
```

**禁止**：
```rust
// ❌ 不要在 gateway/mod.rs 逐个注册
.route("/api/agents", get(huanxing::agents::list))
.route("/api/agents", post(huanxing::agents::create))
.route("/api/sessions", get(huanxing::sessions::list))
// ... 10+ 行
```

### 规则 3：配置用独立 section

**config.toml 结构**：
```toml
# ZeroClaw 原版配置
[llm]
provider = "openai_compat"

# 唤星扩展配置（独立 section）
[huanxing]
enabled = true
api_base_url = "https://api.huanxing.dcfuture.cn"
title_model = "gpt-4o-mini"
```

**Rust 结构**：
```rust
// src/huanxing/config.rs
#[derive(Debug, Clone, Serialize, Deserialize, Default)]
pub struct HuanxingExtConfig {
    pub enabled: bool,
    pub api_base_url: String,
    pub title_model: String,
    // 所有唤星特有配置都在这里
}
```

### 规则 4：State 用 Extension，不改 AppState

**允许**：
```rust
// 用 axum Extension 注入唤星状态
#[cfg(feature = "huanxing")]
let app = app.layer(Extension(HuanxingState { ... }));

// handler 中提取
async fn handler(Extension(hx): Extension<HuanxingState>) { ... }
```

**禁止**：
```rust
// ❌ 不要在上游 AppState 加字段
pub struct AppState {
    pub agent_pool: Arc<AgentContextPool>,  // ❌ 唤星特有
    pub tenant_db: Arc<TenantDb>,           // ❌ 唤星特有
}
```

### 规则 5：渠道逻辑移到 huanxing/channels/

**允许**（channels/mod.rs 只加 feature gate）：
```rust
#[cfg(feature = "huanxing")]
pub mod napcat;

#[cfg(feature = "huanxing")]
pub use napcat::NapcatChannel;
```

**渠道实现放在**：
```
src/huanxing/channels/
├── mod.rs
├── napcat.rs      # QQ NapCat
├── wechat.rs      # 微信
└── ...
```

## 桌面端开发规范

### 前端独立原则

桌面端 `clients/desktop/` 完全独立于 `web/` 目录：

| 配置项 | 指向 |
|--------|------|
| vite alias `@` | `./src`（不是 `../../web/src`） |
| tsconfig paths | `./src/*` |
| Tailwind @source | `.`（扫描自身） |
| node_modules | 独立安装 |

### 组件复用方式

需要复用 web 组件时，**复制到桌面端目录**，不要 import：

```bash
# ✅ 正确：复制到桌面端
cp web/src/components/ChatMessage.tsx clients/desktop/src/components/

# ❌ 错误：跨目录 import
import ChatMessage from "../../../web/src/components/ChatMessage"
```

### Tauri Sidecar 架构

桌面端的 zeroclaw 不是独立进程，由 Tauri 管理：

```
用户登录 → 获取 llm_token → Tauri 创建 sidecar → sidecar 使用独立配置/端口
```

- 配置文件：`~/.zeroclaw/desktop/config.toml`（与 CLI 版本隔离）
- 端口：动态分配（如 42620），不是固定的 42617
- 生命周期：跟随 Tauri 窗口

## 编译与测试

### 编译命令

```bash
# ZeroClaw 原版（无唤星功能）
cargo build

# 唤星桌面端
cargo build --features huanxing

# 唤星全功能
cargo build --features "huanxing,channel-lark"

# Release 版本
cargo build --release --features huanxing
```

### 桌面端开发

```bash
cd clients/desktop

# 前端开发（需要 sidecar 运行）
pnpm dev

# Tauri 开发（自动启动 sidecar）
pnpm tauri dev

# 构建
pnpm tauri build
```

### 代码检查

```bash
# 格式化 + lint
cargo fmt --all -- --check
cargo clippy --all-targets --features huanxing -- -D warnings

# 测试
cargo test --features huanxing
```

## Git 工作流

### 分支策略

| 分支 | 用途 |
|------|------|
| `main` | 稳定版本 |
| `huanxing-clean` | 上游合并工作分支 |
| `feature/*` | 功能开发 |

### 上游同步流程

```bash
# 1. 获取上游更新
git fetch upstream
git checkout huanxing-clean
git merge upstream/master

# 2. 解决冲突（应该只有少量 cfg(feature) 行）
# 3. 测试
cargo build --features huanxing
pnpm tauri dev

# 4. 合并到 main
git checkout main
git merge huanxing-clean
```

### Commit 规范

```
<type>(<scope>): <description>

type: feat/fix/refactor/docs/test/chore
scope: desktop/huanxing/hasn/gateway/...
```

示例：
```
feat(desktop): 桌面端断开 web 目录依赖
fix(huanxing): 修复 Agent 创建时的权限检查
refactor(hasn): 消息同步逻辑移到 hasn-client-core
```

## 冲突预防检查清单

提交前检查：

- [ ] 上游文件只添加了 `#[cfg(feature = "huanxing")]` 行
- [ ] 没有在上游 struct 中直接添加字段
- [ ] 新路由在 `src/huanxing/router.rs` 中注册
- [ ] 新配置在 `HuanxingExtConfig` 中定义
- [ ] 桌面端代码在 `clients/desktop/` 中，没有修改 `web/`
- [ ] 运行 `cargo build` （无 feature）仍然能编译通过

## 常见问题

### Q: 需要修改上游的某个函数签名怎么办？

A: 优先考虑：
1. 用 wrapper 函数包装
2. 用 trait extension 扩展
3. 提 PR 给上游

实在不行才直接改，并在代码中标注 `// HUANXING: 修改原因`

### Q: 上游更新后编译失败怎么办？

A: 检查：
1. 上游 API 签名是否变化 → 更新 `src/huanxing/` 中的调用
2. 上游 struct 字段是否变化 → 更新 `HuanxingExtConfig`
3. 依赖版本是否冲突 → 检查 `Cargo.toml`

### Q: 桌面端缺少某个 npm 包怎么办？

A: 在 `clients/desktop/package.json` 中添加，不要依赖 `web/node_modules`
